# 05 · 开发分步计划

> 版本：v1.0 · 日期：2026-06-24

---

## 里程碑总览

| 阶段 | 名称 | 预计占比 | 交付物 |
|------|------|------|------|
| M1 | 项目初始化 | ~10% | 目录骨架、依赖安装、SQL 脚本输出、环境配置（数据库由用户自行构建） |
| M2 | 后端核心接口 | ~25% | 全部 REST API 可用（Postman 验证通过） |
| M3 | 前端公共层 + 首页 | ~15% | 导航栏/页脚、公共样式、API 封装、首页可用 |
| M4 | 列表页 + 详情页 | ~15% | 搜索/筛选/排序/分页 + 图书详情 + 收藏 |
| M5 | 用户系统 | ~15% | 注册/登录 + JWT 集成 + 个人中心 + 收藏列表 |
| M6 | 后台管理 | ~10% | 管理页 CRUD + 权限控制 |
| M7 | 收尾与文档 | ~10% | README、AI使用说明、最终联调、数据校验 |

---

## M1 · 项目初始化

**目标：** 目录骨架就绪，后端能启动，SQL 脚本已交付用户。

**任务清单：**

1. 按 `03-项目目录结构.md` 创建全部目录和空文件
2. `backend/package.json`：`npm init` + 安装全部依赖
3. `backend/.env.example`：编写环境变量模板（DB_HOST, DB_USER, DB_PASS, DB_NAME, JWT_SECRET, PORT）
4. `backend/src/config/db.js`：mysql2 连接池（使用 `.env` 中的连接信息）
5. `backend/src/server.js`：Express 最小骨架（cors + json + 端口监听）
6. `database/schema.sql`：CREATE DATABASE + 4 张表
7. `database/seed.sql`：3 分类 + ~15 本图书 + 1 admin + 1 普通用户
8. **以上 SQL 脚本交付用户自行执行**；用户建库完成后提供 DB_HOST / DB_USER / DB_PASS / DB_NAME，回填至 `backend/.env`

**交付物：** 后端 `npm start` 不报错（待用户提供数据库连接信息后可正常连接）；`database/` 下两份 SQL 脚本就位。

---

## M2 · 后端核心接口

**目标：** 全部 API 实现并通过 Postman 手动验证。

**任务清单：**

1. `backend/src/utils/response.js`：统一响应工具函数
2. `backend/src/middleware/auth.js`：JWT 验证中间件 + admin 角色中间件
3. `backend/src/routes/books.js`：B1–B6
4. `backend/src/routes/auth.js`：U1–U2
5. `backend/src/routes/user.js`：U3
6. `backend/src/routes/favorites.js`：F1–F3
7. `backend/src/routes/categories.js`：C1
8. `server.js` 注册所有路由
9. 使用 Postman 对每个接口逐一验证（正常 + 异常场景）

**交付物：** 全部 13 个接口可正常响应，JSON 格式符合 `04-API接口规范.md`。

---

## M3 · 前端公共层 + 首页

**目标：** 首页完整可用（推荐图书 + 学生信息）。

**任务清单：**

1. `frontend/css/common.css`：导航栏、页脚、按钮、表单、卡片、布局栅格
2. `frontend/js/config.js`：API_BASE_URL
3. `frontend/js/api.js`：fetch 封装（GET/POST/PUT/DELETE，自动带 JWT，统一错误处理）
4. `frontend/js/auth.js`：登录状态检查、token 读取/存储、登出
5. `frontend/js/common.js`：导航栏渲染（根据登录状态+角色显示不同菜单）、页脚渲染
6. `frontend/index.html`：页面结构
7. `frontend/css/index.css`：首页样式
8. `frontend/js/index.js`：调用推荐接口 + 渲染卡片 + 右上角学生信息
9. 放置 `frontend/images/photo.jpg` 和约 15 张封面图到 `frontend/images/covers/`

**交付物：** 打开 `index.html` 可见推荐图书卡片 + 右上角学生照片/姓名/学号。

---

## M4 · 列表页 + 详情页

**目标：** 图书浏览、搜索、筛选、排序、分页全部可用；详情页可查看完整信息。

**任务清单：**

1. `frontend/list.html` + `frontend/css/list.css` + `frontend/js/list.js`
   - 卡片网格、搜索框、分类筛选下拉、排序切换、分页器
2. `frontend/detail.html` + `frontend/css/detail.css` + `frontend/js/detail.js`
   - 全字段展示、收藏/取消收藏按钮（需登录）
3. 列表页到详情页的跳转联动（`?id=xx`）

**交付物：** 可浏览 15 本图书，可按分类筛选、关键词搜索、评分排序、翻页；点击可进入详情页。

---

## M5 · 用户系统

**目标：** 注册、登录、个人中心、收藏功能完整闭环。

**任务清单：**

1. `frontend/login.html` + `frontend/css/auth.css` + `frontend/js/login.js`
2. `frontend/register.html` + `frontend/js/register.js`
3. `frontend/profile.html` + `frontend/css/profile.css` + `frontend/js/profile.js`
   - 用户信息展示 + 收藏列表 + 取消收藏
4. 全站导航栏登录状态联动（登录后显示用户名+"管理"入口[管理员]+"个人中心"；未登录显示"登录/注册"）
5. 收藏按钮在详情页和列表页的状态同步

**交付物：** 可注册新用户 → 登录 → 收藏图书 → 个人中心查看收藏 → 取消收藏。管理员登录后导航栏出现"管理"入口。

---

## M6 · 后台管理

**目标：** 管理员可增删改查图书。

**任务清单：**

1. `frontend/admin.html` + `frontend/css/admin.css` + `frontend/js/admin.js`
   - 图书表格列表
   - 新增表单（模态框或独立区域）
   - 编辑表单（数据回填）
   - 删除按钮 + 二次确认
2. 非管理员访问时重定向或提示无权限

**交付物：** 管理员可新增图书 → 出现在列表中 → 可编辑 → 可删除；普通用户无法访问管理页。

---

## M7 · 收尾与文档

**目标：** 项目完整可提交。

**任务清单：**

1. `README.md`：项目名称、主题、主要功能、运行方式、数据库说明、测试账号
2. `AI使用说明.md`：按课程要求格式填写
3. `backend/.env.example` 内容完整，`.env` 已加入 `.gitignore`
4. 全流程联调：从前端页面出发，走完所有功能路径
5. 边界情况检查：空状态（无搜索结果、无收藏）、Token 过期、重复注册等
6. 代码清理：删除注释掉的废代码、console.log（调试用的）、未使用的变量

**交付物：** 项目完整可运行，文档齐全，满足课程全部要求。

---

```
  └→ M2（后端接口）
        └→ M3（公共层+首页）──→ M4（列表+详情）──→ M5（用户系统）──→ M6（后台管理）
                                                                    └→ M7（收尾）
```
