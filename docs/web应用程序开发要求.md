
## 项目主题要求

图书推荐网站

项目必须是一个完整网站，不能只做单个静态页面。

项目必须包含：

前端页面
后端接口
数据库数据
前后端交互

## 技术要求

前端技术：

HTML
CSS
JavaScript

后端技术：

Node.js
Express

数据库：
MySQL

## 页面基本要求

网站至少应包含以下页面：

1. 首页
2. 列表页
3. 详情页
4. 登录页或注册页
5. 后台管理页或个人中心页

其中，首页必须满足以下要求：

页面右上角固定显示本人照片。
照片下方显示本人姓名和学号。

示例：
右上角：
学生照片
姓名：张三
学号：20240001

要求：该信息应固定在首页右上角，不能隐藏在不明显位置。


## 功能基本要求

项目至少应完成以下功能。

### 1. 前台展示功能

应包含：

```txt
首页展示推荐内容
列表页展示多条数据
详情页展示单条数据详情
```

例如：

博客类网站：

```txt
首页展示推荐文章
列表页展示文章列表
详情页展示文章全文
```

新闻类网站：

```txt
首页展示热点新闻
列表页展示新闻列表
详情页展示新闻正文
```

校园活动类网站：

```txt
首页展示近期活动
列表页展示活动列表
详情页展示活动介绍
```

### 2. 数据查询功能

至少实现以下一种查询功能：

```txt
关键词搜索
分类筛选
按时间排序
按类型查询
```

### 3. 用户相关功能

至少完成以下一种：

```txt
用户注册
用户登录
个人中心
我的收藏
我的报名
我的发布
```

如实现登录功能，建议使用：

```txt
localStorage
JWT
后端登录校验
```

### 4. 后台管理功能

至少完成一种数据管理功能：

```txt
新增数据
修改数据
删除数据
查看数据
```

例如：

博客类网站：

```txt
新增文章
编辑文章
删除文章
```

新闻类网站：

```txt
发布新闻
编辑新闻
删除新闻
```

校园活动类网站：

```txt
发布活动
编辑活动
删除活动
```


## 数据库要求

项目必须使用 MySQL 保存数据。

至少应包含：

```txt
1 张主要业务表
```

建议包含：

```txt
users 用户表
articles / news / activities / products 等业务表
```

数据库表设计应与项目主题相关。

示例：

博客类网站：

```txt
users
articles
comments
```

新闻类网站：

```txt
users
news
categories
```

校园活动平台：

```txt
users
activities
registrations
```

数据库中应有一定数量的测试数据，不能只展示一条数据。

## 接口要求

后端必须使用 Express 提供接口。

至少应包含：

```txt
GET 查询接口
POST 新增或登录接口
```

建议包含：

```txt
GET    查询列表
GET    查询详情
POST   新增数据
PUT    修改数据
DELETE 删除数据
POST   登录或注册
```

接口应能被前端页面正常调用。

前端不能只写静态假数据，应至少有一部分页面数据来自后端接口和 MySQL 数据库。


## AI 辅助编程使用要求

本次考试允许使用 AI 辅助编程，但必须遵守以下要求：

1. 可以使用 AI 帮助生成代码、排查错误、解释语法、优化样式。
2. 不得直接提交完全无法解释的代码。
3. 使用 AI 的部分必须在项目说明文档中声明。
4. 考试结束时，教师可以要求学生现场解释关键代码。

项目中必须提交 `AI使用说明.md`。

内容格式如下：

```md
# AI 使用说明

## 一、是否使用 AI

是 / 否

## 二、使用 AI 的位置

1. 使用 AI 辅助设计页面布局。
2. 使用 AI 辅助生成部分 CSS 样式。
3. 使用 AI 辅助排查后端接口错误。

## 三、本人完成的内容

1. 项目主题设计。
2. 数据库表创建。
3. 页面代码整合。
4. 前后端联调。

## 四、能够独立说明的关键代码

1. Express 路由。
2. MySQL 查询。
3. 前端 fetch 请求。
4. DOM 渲染。
```
