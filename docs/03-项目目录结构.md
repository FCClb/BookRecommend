## 二、目录职责速查

| 目录 | 职责 |
|------|------|
| `frontend/` | 所有前端静态文件，无构建步骤，可直接通过浏览器或 Live Server 访问 |
| `frontend/css/` | 样式文件，`common.css` 为全站公共样式，其余按页面独立拆分 |
| `frontend/js/` | 脚本文件，`config.js` + `api.js` + `auth.js` + `common.js` 为公共基础层 |
| `frontend/images/` | 图片资源：学生照片 + 图书封面 |
| `backend/` | Express 后端应用，通过 `npm start` 启动 |
| `backend/src/` | 后端源码，入口 `server.js`，路由分层 |
| `backend/src/config/` | 数据库连接池配置 |
| `backend/src/routes/` | 路由模块，按资源拆分（books / auth / favorites / categories / user） |
| `backend/src/middleware/` | Express 中间件（JWT 鉴权、角色校验） |
| `backend/src/utils/` | 通用工具函数（统一响应格式） |
| `database/` | SQL 脚本，一次性执行即可建库建表 + 填充测试数据 |
| `docs/` | 项目规范与开发文档 |

## 三、命名约定

| 类型 | 约定 | 示例 |
|------|------|------|
| HTML 文件 | 小写短横线或功能名 | `list.html`, `profile.html` |
| CSS 文件 | 与页面同名，`common.css` 为公共 | `list.css`, `admin.css` |
| JS 文件 | 与页面同名，公共层集中管理 | `api.js`, `auth.js` |
| 路由文件 | 资源名复数 | `books.js`, `categories.js` |
| SQL 文件 | 语义化命名 | `schema.sql`, `seed.sql` |
| 封面图片 | `book-` + 两位序号 + `.jpg` | `book-01.jpg` ~ `book-15.jpg` |
