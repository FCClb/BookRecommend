## 二、数据库构建

### 2.1 创建数据库

```bash
# 在 MySQL 命令行中依次执行
mysql -u root -p < database/schema.sql
mysql -u root -p < database/seed.sql
```

> `schema.sql` 创建 `book_recommend` 数据库和 4 张表。
> `seed.sql` 插入 3 个分类、2 个测试用户、15 本图书。

### 2.2 验证数据库

```sql
USE book_recommend;
SELECT COUNT(*) FROM categories;   -- 应返回 3
SELECT COUNT(*) FROM users;        -- 应返回 2
SELECT COUNT(*) FROM books;        -- 应返回 15
```

## 三、后端启动

> **注意：** 以下所有命令均在 `backend/` 目录下执行，不是在项目根目录。

### 3.1 进入后端目录

```bash
cd backend
```

### 3.2 安装依赖

```bash
npm install
```

### 3.3 配置环境变量

编辑 `backend/.env`（与 `package.json` 同级），填入实际值：

```env
DB_HOST=localhost
DB_USER=root
DB_PASS=你的MySQL密码
DB_NAME=book_recommend
JWT_SECRET=随便写一串英文字母数字
PORT=3000
```

### 3.4 启动服务

```bash
npm start
```

看到 `✅ 服务已启动: http://localhost:3000` 即为成功。

### 3.4 验证后端

浏览器打开 `http://localhost:3000`，应返回：

```json
{"code":200,"message":"图书推荐网站 API 运行中","data":null}
```

## 四、前端启动

### 4.1 使用 VS Code Live Server

1. 在 VS Code 中打开 `frontend/` 目录
2. 右键 `index.html` → **Open with Live Server**
3. 浏览器自动打开 `http://127.0.0.1:5500`

> 或者用其他静态服务器（如 nginx、http-server），只要把 `frontend/` 作为根目录即可。
> 前端默认连接 `http://localhost:3000` 后端（配置在 `frontend/js/config.js`）。

### 4.2 确认封面图片

确保 `frontend/images/covers/` 下有 15 张封面图（`book-01.jpg` ~ `book-15.jpg`），否则图书卡片显示书名首字符占位符。

## 六、API 接口快速测试（curl）

以下命令在 Git Bash 终端执行：

### 6.1 公开接口（无需登录）

```bash
# 根路径
curl http://localhost:3000/

# 首页推荐
curl http://localhost:3000/api/books/recommended

# 图书列表（含搜索/筛选/排序）
curl "http://localhost:3000/api/books?keyword=三体"
curl "http://localhost:3000/api/books?category_id=1&sort=rating_desc"
curl "http://localhost:3000/api/books?page=2"

# 图书详情
curl http://localhost:3000/api/books/1

# 分类列表
curl http://localhost:3000/api/categories
```

### 6.2 注册 / 登录

```bash
# 注册新用户
curl -X POST http://localhost:3000/api/auth/register \
  -H "Content-Type: application/json" \
  -d '{"username":"apitest","password":"123456"}'

# 重复注册（应返回 409）
curl -X POST http://localhost:3000/api/auth/register \
  -H "Content-Type: application/json" \
  -d '{"username":"apitest","password":"123456"}'

# 登录获取 Token
curl -X POST http://localhost:3000/api/auth/login \
  -H "Content-Type: application/json" \
  -d '{"username":"apitest","password":"123456"}'

# 错误密码登录（应返回 401）
curl -X POST http://localhost:3000/api/auth/login \
  -H "Content-Type: application/json" \
  -d '{"username":"apitest","password":"wrong"}'
```

### 6.3 认证接口（需替换 TOKEN）

```bash
# 先登录获取 TOKEN
TOKEN=$(curl -s -X POST http://localhost:3000/api/auth/login \
  -H "Content-Type: application/json" \
  -d '{"username":"testuser","password":"123456"}' | grep -o '"token":"[^"]*"' | cut -d'"' -f4)

# 用户信息
curl http://localhost:3000/api/user/profile \
  -H "Authorization: Bearer $TOKEN"

# 收藏列表
curl http://localhost:3000/api/favorites \
  -H "Authorization: Bearer $TOKEN"

# 添加收藏
curl -X POST http://localhost:3000/api/favorites \
  -H "Authorization: Bearer $TOKEN" \
  -H "Content-Type: application/json" \
  -d '{"book_id":1}'

# 重复收藏（应返回 409）
curl -X POST http://localhost:3000/api/favorites \
  -H "Authorization: Bearer $TOKEN" \
  -H "Content-Type: application/json" \
  -d '{"book_id":1}'

# 取消收藏
curl -X DELETE http://localhost:3000/api/favorites/1 \
  -H "Authorization: Bearer $TOKEN"

# 未登录访问（应返回 401）
curl http://localhost:3000/api/user/profile
```

### 6.4 管理员接口（需 admin Token）

```bash
# 管理员登录获取 TOKEN
ADMIN_TOKEN=$(curl -s -X POST http://localhost:3000/api/auth/login \
  -H "Content-Type: application/json" \
  -d '{"username":"admin","password":"admin123"}' | grep -o '"token":"[^"]*"' | cut -d'"' -f4)

# 新增图书
curl -X POST http://localhost:3000/api/books \
  -H "Authorization: Bearer $ADMIN_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{"title":"测试图书","author":"测试作者","category_id":2,"rating":8.0}'

# 修改图书（替换 BOOK_ID 为新增返回的 id）
curl -X PUT http://localhost:3000/api/books/BOOK_ID \
  -H "Authorization: Bearer $ADMIN_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{"rating":8.5}'

# 删除图书
curl -X DELETE http://localhost:3000/api/books/BOOK_ID \
  -H "Authorization: Bearer $ADMIN_TOKEN"

# 普通用户访问管理接口（应返回 403）
USER_TOKEN=$(curl -s -X POST http://localhost:3000/api/auth/login \
  -H "Content-Type: application/json" \
  -d '{"username":"testuser","password":"123456"}' | grep -o '"token":"[^"]*"' | cut -d'"' -f4)
curl -X POST http://localhost:3000/api/books \
  -H "Authorization: Bearer $USER_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{"title":"hack"}'
```

## 七、错误场景测试清单

| 编号 | 场景 | 触发方式 | 预期 |
|------|------|----------|------|
| E1 | 访问不存在的图书详情 | `detail.html?id=999` | 显示"加载失败：图书不存在" |
| E2 | 缺少图书 ID | `detail.html`（不带参数） | 显示"缺少图书 ID 参数" |
| E3 | 注册重复用户名 | 注册页输入 `admin` | 红色提示"用户名已存在" |
| E4 | 登录错误密码 | 登录页输入错误密码 | 红色提示"用户名或密码错误" |
| E5 | Token 过期后访问 | 清除 localStorage 的 token，访问 `profile.html` | 跳转到登录页 |
| E6 | 普通用户访问后台 | 用 `testuser` 登录，访问 `admin.html` | 显示"权限不足" |
| E7 | 非管理员调用管理 API | curl 用普通用户 Token 调 POST /api/books | 返回 403 |
| E8 | 重复收藏同一本书 | 已收藏的书再次点收藏 | Toast 提示"已收藏过该书" |
| E9 | 后端未启动时访问前端 | 停止 `npm start`，刷新页面 | 列表/推荐区域显示"加载失败：无法连接到服务器" |
| E10 | 封面图缺失 | 删除某本书的封面文件 | 对应卡片显示书名首字符占位符 |
