# 02 · 技术选型文档

> 版本：v1.0 · 日期：2026-06-24

---

## 一、选型总览

| 层 | 技术 | 版本 | 选型理由 |
|------|------|------|----------|
| **前端框架** | 原生 HTML + CSS + JS | — | 课程要求，无框架依赖，降低复杂度 |
| **后端运行时** | Node.js | 18.x / 20.x LTS | 课程要求，事件驱动，生态丰富 |
| **后端框架** | Express | 4.21.x | 课程要求，最主流 Node.js Web 框架，轻量灵活 |
| **数据库** | MySQL | 8.0.x | 课程要求，关系型数据，适合图书/用户/分类建模 |
| **数据库驱动** | mysql2 | 3.11.x | Promise 原生支持，比 mysql 包性能更好、更安全 |
| **密码哈希** | bcryptjs | 2.4.x | 纯 JS 实现，免 Node 原生编译依赖，跨平台零配置 |
| **身份认证** | jsonwebtoken | 9.0.x | JWT 标准实现，轻量无状态，适合前后端分离 |
| **环境变量** | dotenv | 16.4.x | 将 DB 密码等敏感配置抽离 .env，零依赖侵入 |
| **跨域** | cors | 2.8.x | 前端独立文件需要通过跨域访问后端 API |
| **输入验证** | express-validator | 7.2.x | 链式校验请求体、参数，防注入与脏数据 |
| **开发工具** | nodemon | 3.1.x | 文件变更自动重启后端，提升开发效率 |

---

## 二、不引入的技术（及原因）

| 未选技术 | 原因 |
|----------|------|
| **React / Vue / Angular** | 课程要求原生 JS，且项目规模小，不需要 MVVM 框架 |
| **TypeScript** | 课程要求 JavaScript，增加编译步骤不适合课程场景 |
| **ORM（Sequelize / Prisma）** | 表少、查询简单，原生 SQL 更透明、易解释 |
| **Webpack / Vite** | 原生 JS 无打包需求，直接通过 Live Server 或静态托管运行 |
| **Redis** | 无缓存/会话集中存储需求，JWT 已满足无状态认证 |
| **文件上传（multer）** | 封面图使用本地静态文件，不需要上传功能 |

---

## 三、数据库选型补充说明

选用 **MySQL 8.0**：

- 课程明确要求
- 本项目实体关系简单（users → favorites ← books ← categories），关系型模型天然匹配
- `mysql2` 驱动支持 `async/await`，与 Express 中间件风格一致
- 字符集统一使用 `utf8mb4`，排序规则 `utf8mb4_unicode_ci`

---

## 四、前端依赖说明

前端不使用 npm 依赖，所有功能以原生浏览器 API 实现：

| 能力 | 实现方式 |
|------|----------|
| HTTP 请求 | `fetch()` API |
| DOM 操作 | `document.querySelector` 等原生 API |
| 路由/参数 | `URLSearchParams` 获取查询字符串 |
| Token 存储 | `localStorage` |
| 模板渲染 | 模板字符串 + `innerHTML`（非 innerHTML 拼接，需防范 XSS） |
| CSS | 原生 CSS（Flexbox + Grid），无预处理器 |
