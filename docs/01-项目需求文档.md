# 01 · 项目需求文档

> 版本：v1.0 · 状态：已确认 · 日期：2026-06-24

---

## 一、产品概述

| 项 | 内容 |
|----|------|
| **项目名称** | 图书推荐网站 |
| **一句话描述** | 以图书推荐为核心的完整前后端网站，用户可浏览、搜索、收藏图书，管理员可管理图书数据 |
| **技术栈** | 前端 HTML + CSS + JS（原生）；后端 Node.js + Express；数据库 MySQL |
| **目标设备** | PC 桌面端（1920×1080 / 1366×768，不做移动端响应式） |
| **学生信息** | 高锦博 · 24469209 |

---

## 二、用户角色与权限模型

| 角色 | 权限说明 |
|------|----------|
| **普通用户（user）** | 浏览首页推荐、图书列表、搜索/筛选/排序、查看详情、注册、登录、个人中心、收藏/取消收藏图书 |
| **管理员（admin）** | 普通用户全部权限 + 进入后台管理页 + 图书新增/编辑/删除 |

- 导航栏"管理"入口仅对已登录管理员可见
- 后台接口须校验 admin 角色，直接 URL 访问也须鉴权

---

## 三、功能清单

### 3.1 前台展示

| 编号 | 功能 | 详细说明 |
|------|------|----------|
| F1 | 首页推荐 | 按**评分从高到低**展示推荐图书 |
| F2 | 首页学生信息 | 页面右上角固定显示：学生照片 + 姓名 + 学号，不被任何元素遮挡 |
| F3 | 图书列表页 | 全部图书以卡片网格展示，每页 **8 条**，支持搜索、筛选、排序 |
| F4 | 图书详情页 | 展示单本图书全部字段信息 |
| F5 | 关键词搜索 | 按**书名 + 作者**模糊匹配 |
| F6 | 分类筛选 | 按固定分类筛选 |
| F7 | 评分排序 | 按评分升序 / 降序排列 |

### 3.2 用户功能

| 编号 | 功能 | 说明 |
|------|------|------|
| U1 | 注册 | 输入用户名 + 密码，后端 bcrypt 加密后存库 |
| U2 | 登录 | 输入用户名 + 密码，后端校验后返回 JWT token，前端存 localStorage |
| U3 | 个人中心 | 展示当前用户信息 + 我的收藏列表 |
| U4 | 收藏图书 | 详情页/列表页可收藏；已收藏可取消；个人中心展示收藏列表 |

### 3.3 后台管理（仅管理员）

| 编号 | 功能 | 说明 |
|------|------|------|
| A1 | 新增图书 | 管理员填写表单，包含全部图书字段 |
| A2 | 编辑图书 | 管理员修改已有图书 |
| A3 | 删除图书 | 管理员删除图书（需二次确认） |
| A4 | 查看图书列表 | 管理端表格展示全部图书 |

## 四、数据实体

### 4.1 用户表 `users`

| 字段 | 类型 | 约束 | 说明 |
|------|------|------|------|
| id | INT | PK, AUTO_INCREMENT | 用户 ID |
| username | VARCHAR(50) | NOT NULL, UNIQUE | 登录用户名 |
| password | VARCHAR(255) | NOT NULL | bcrypt 哈希 |
| role | ENUM('user','admin') | NOT NULL, DEFAULT 'user' | 角色 |
| created_at | DATETIME | DEFAULT CURRENT_TIMESTAMP | 注册时间 |

### 4.2 图书表 `books`

| 字段 | 类型 | 约束 | 说明 |
|------|------|------|------|
| id | INT | PK, AUTO_INCREMENT | 图书 ID |
| title | VARCHAR(200) | NOT NULL | 书名 |
| author | VARCHAR(100) | NOT NULL | 作者 |
| cover_url | VARCHAR(500) | — | 封面图相对路径 |
| description | TEXT | — | 内容简介 |
| category_id | INT | FK → categories.id | 所属分类 |
| rating | DECIMAL(3,2) | DEFAULT 0.00 | 评分（0.00–10.00） |
| publisher | VARCHAR(200) | — | 出版社 |
| publish_date | DATE | — | 出版日期 |
| pages | INT | — | 页数 |
| created_at | DATETIME | DEFAULT CURRENT_TIMESTAMP | 入库时间 |
| updated_at | DATETIME | ON UPDATE CURRENT_TIMESTAMP | 更新时间 |

### 4.3 分类表 `categories`

| 字段 | 类型 | 约束 | 说明 |
|------|------|------|------|
| id | INT | PK, AUTO_INCREMENT | 分类 ID |
| name | VARCHAR(50) | NOT NULL, UNIQUE | 分类名称 |

> 固定 3 个分类，由数据库初始化脚本写入，不提供管理界面动态维护。

### 4.4 收藏表 `favorites`

| 字段 | 类型 | 约束 | 说明 |
|------|------|------|------|
| id | INT | PK, AUTO_INCREMENT | 收藏 ID |
| user_id | INT | FK → users.id, NOT NULL | 用户 ID |
| book_id | INT | FK → books.id, NOT NULL | 图书 ID |
| created_at | DATETIME | DEFAULT CURRENT_TIMESTAMP | 收藏时间 |

> UNIQUE KEY (user_id, book_id) 防止重复收藏。

## 六、UI 规范

| 维度 | 规范 |
|------|------|
| **整体风格** | 简洁书卷风，暖色调（米白 #FDF6EC 底 + 棕色 #5D4037 / 墨绿点缀） |
| **布局** | 顶部固定导航栏（Logo + 菜单 + 登录状态）→ 主体内容 → 底部页脚 |
| **首页右上角** | 固定悬浮区域（position: fixed），圆形头像（border-radius: 50%）+ 下方姓名 + 学号，z-index 保证不被遮挡 |
| **图书卡片** | 封面图（固定宽高比）→ 书名（加粗）→ 作者 → 星级评分 → 出版社；hover 时上浮 4px + 阴影加深 |
| **列表页网格** | 每行 4 张卡片（25% 宽），每页 8 本 = 2 行 |
| **封面图** | 存于 `frontend/images/covers/`，数据库存相对路径如 `/images/covers/xxx.jpg` |
| **学生照片** | 存于 `frontend/images/photo.jpg`，项目本地文件 |
| **字体** | 中文：微软雅黑 / PingFang SC；英文：Segoe UI |
| **圆角** | 卡片 8px，按钮 4px |

## 七、数据边界与约束

| 项 | 值 |
|------|------|
| 图书测试数据 | 约 15 本 |
| 分类数量 | 3 个（固定） |
| 列表分页 | 每页 8 条 |
| 管理员初始账号 | 由 seed.sql 预置 |
| 普通用户初始账号 | 由 seed.sql 预置 |
| 密码最小长度 | ≥ 6 位 |
| JWT 过期时间 | 7 天 |

## 八、非功能需求

| 项 | 说明 |
|------|------|
| 安全性 | 密码 bcrypt 哈希；JWT 中间件保护敏感接口；SQL 参数化查询防注入 |
| 数据完整性 | 外键约束；收藏去重唯一索引；空字段前端提示 |
| 可维护性 | 目录结构清晰分层；关键配置抽离 .env；接口规范统一 JSON 响应格式 |
| 浏览器兼容 | Chrome / Edge 最新两版 |
