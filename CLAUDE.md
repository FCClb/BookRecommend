## 协作约束

1. **文档优先**：修改任何业务代码前，必须先查阅 `docs/` 下对应的规范文档，确保实现与文档一致。若发现文档与需求冲突，先提出修订建议，等待确认后再改。
2. **分步推进**：严格遵循 `05-开发分步计划.md` 的里程碑顺序（M1 → M2 → … → M7），不跳跃开发。每个里程碑完成后明确告知用户，等待确认再进入下一里程碑。
3. **统一响应格式**：所有后端接口响应必须遵循 `04-API接口规范.md` 定义的 `{ code, message, data }` 结构。
4. **目录约定**：所有新建文件必须落在 `03-项目目录结构.md` 约定的路径下，不随意创建额外目录。
5. **命名一致**：文件名、路由路径、CSS 类名、JS 函数名遵循 `03-项目目录结构.md` 的命名约定。
6. **安全底线**：密码必须 bcrypt 哈希存储（`02-技术选型文档.md`）；SQL 使用参数化查询；敏感配置写入 `.env`。
7. **最小依赖**：不引入 `02-技术选型文档.md` 中"不引入的技术"列表中的任何包或工具，除非用户明确要求。
8. **数据库分工**：只输出 SQL 脚本（`database/schema.sql` + `database/seed.sql`），不直接连接或操作用户的 MySQL 实例。数据库由用户自行构建，用户完成后提供连接信息（DB_HOST / DB_USER / DB_PASS / DB_NAME），回填至 `backend/.env`。
